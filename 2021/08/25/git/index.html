git 常用操作和命令

发表于2021-08-25|更新于2021-08-31|编程
|阅读量:

唠唠闲话

Git 是目前世界上最先进的分布式版本控制系统，可以用于敏捷高效地处理任何或小或大的项目。
个人而言，主要用 Git + Github 实现这些功能：

  • 备份，同步本地项目
  • 多设备管理项目
  • 文件历史版本（改错的后悔药）
  • 克隆 Github 项目到本地
  • 部署博客网站

本篇主要目的是记录 Git 中经常用的操作和命令，方便查阅，至于 Git 的安装方法和工作原理网上有大把教程，就没必要说了。

注：初学 Git 要理解三个概念： 工作区，缓存区 和 版本库。
推荐网站：廖学锋的 Git 教程，最开始学习 Git 就在这个网站。

工作区操作

  1. 回退暂存区到 HEAD 状态

    特别注意：使用 git restore 时，如果缺少参数 --staged ，执行内容时是恢复工作区的修改为 HEAD 状态，这直接会导致未保存的修改被撤销，谨慎使用。

  2. 删除/清空暂存区
    git rm --cached <file> -r

    注：<file> 指定从暂存区删除的文件，用 . 可将暂存区清空，清空目录需要加参数 -r。

版本回退

使用 git reset 版本回退，改变 HEAD 指向的位置。有三个可选参数 soft, hard 和 mixed，不同参数对工作区和暂存区有不同影响，默认参数为 mixed，参考这里。
三者区别如下:

  1. git reset —soft
    保留 工作区和暂存区 的内容，简单来说就是你的代码还在只是变成了未提交状态或未添加状态
  2. git reset —hard
    回退 工作区和暂存区 到指定的 commit 版本
  3. git reset —mixed
    保留 工作区 的内容，回退 暂存区 到指定的 commit 版本

    注：git reset 操作中，没有被 git 追踪的文件不会影响

gitignore

git 仓库中，.gitignore 指定的文件和文件夹会被 git add 自动忽略，但 -f 参数可以让 git add 添加被忽略的文件。
使用说明：

  • <file/folder> 忽略所有目录下的该文件或文件夹
  • ./<file/folder> 忽略当前目录下的该文件名或文件夹
  • ! <file/folder ! 代表取消忽略

注：如果文件夹被 .gitignore 忽略，用 git add <path> -f 强制添加后，只有添加时刻的文件被 git 记录，之后文件夹内的其他文件变化都不会被记录。

Github 与 Git

关于 github

GitHub 是世界上最大的同性交友网站基于 Git 的一个代码托管网站。开发者可以将代码在 GitHub 上开源，可以浏览其它项目的代码，fork 到自己名下做修改，clone 回本地（没有访问权限的 private repo 除外）使用，也可以发起 pull request 向上游提交自己的修改。

注：基于 Git 的代码托管网站还有很多，比如 Gitlab， Gitee（码云） 等等，但相比之下，Github 有更多的生态基础。引用 Coding 的一段精炼介绍：Github 发明了碉堡了的两个功能：Fork 和 Pull Request。这两个功能创造了整个 Github 生态系统，使得“基因”得以繁衍和进化，充满了生命力。基因通过 Fork 被复制，而 Pull Request 使得基因得以进化。好的基因会被大量的 Fork，从而实现了优胜劣汰。这一整套体系才是精华所在，说 Github 是代码仓库显然太肤浅了。

远端仓库克隆到本地

  1. 复制远端仓库地址

更新和修改

借助 Github 可在多个设备上同步管理一个项目。假设设备 A 提交了修改，此时远端仓库比设备 B 领先了一个提交。若设备 B 也做了一些修改，直接提交会导致矛盾，需要进行下边步骤再提交。

  1. git commit 保存本地修改
  2. git pull origin master 将远端的修改拉到本地，提示两仓库的差别，在修改合并后提交到本地修改
  3. 此时可以再使用 git push 就能提交了
